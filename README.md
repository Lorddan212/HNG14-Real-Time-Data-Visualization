# Real-Time Operations Intelligence Dashboard

A production-minded Vue 3 + TypeScript real-time analytics dashboard for streaming operational telemetry. The app simulates infrastructure, security, traffic, and regional activity data with responsive ECharts visualizations, centralized Pinia state, resilient validation, and an optimized live activity feed.

## Features

- Mocked WebSocket-style real-time stream with pause, resume, offline, and reconnect simulation.
- Connection status states: connected, reconnecting, offline, and paused.
- Payload validation before state updates, including safe rejection of malformed stream messages.
- Live CPU, memory, network, active user, request, error, latency, service, health, and regional activity signals.
- ECharts line, area, bar, and radar visualizations with tooltips, smooth updates, responsive resizing, and dataset toggles.
- TanStack Table-powered activity feed with search, severity filtering, newest-first ordering, and bounded history.
- Time-range controls for last 1 minute, 5 minutes, 15 minutes, and 1 hour.
- Light dashboard by default with dark mode support.
- Loading, empty, and error states for resilient operation.
- Responsive layout for desktop, tablet, and mobile.

## Data Streaming Approach

`src/services/streamService.ts` provides a WebSocket-like subscription API. It emits unknown payloads, connection status changes, and transport notices. Valid payloads are generated by `dataGenerator.ts`; malformed payloads are occasionally emitted to exercise validation and resilience.

`src/composables/useStreaming.ts` owns subscription setup and cleanup. Incoming messages are batched through `requestAnimationFrame`, validated with `validateStreamPayload`, and only then committed to Pinia. Unsubscribes, timers, and animation frames are cleaned up on unmount.

## State Management Strategy

Pinia keeps state centralized and extensible:

- `dashboardStore` owns metric series, services, health scores, regions, activity events, filters, dataset visibility, chart mode, and loading/error state.
- `streamStore` owns connection status, heartbeat, reconnect attempts, stream errors, and rejected payload records.
- `themeStore` owns light/dark mode and applies the selected theme to the document root.

Derived data such as filtered events and time-range metric series is exposed through Pinia getters so components stay focused on rendering.

## Rendering And Performance Decisions

- Metric series are capped at `MAX_SERIES_POINTS` to avoid unbounded memory growth.
- Activity logs are capped at `MAX_ACTIVITY_ITEMS`, and rendered rows are limited with `MAX_VISIBLE_TABLE_ROWS`.
- Stream updates are batched with `requestAnimationFrame` to avoid unnecessary re-renders during bursts.
- ECharts uses stable chart components, `lazyUpdate`, hidden symbols for dense line charts, smooth update animation, and bounded datasets.
- The activity feed uses TanStack Table for a scalable table model and a scroll container instead of expanding the page indefinitely.
- Computed selectors keep derived chart/table data out of templates.
- Components clean up intervals, listeners, animation frames, and timers.

## Error Handling And Resilience

- Malformed payloads are rejected before reaching dashboard state.
- Rejected payloads are recorded in the stream hygiene panel.
- Connection interruptions trigger reconnect status and backoff simulation.
- Loading, empty, and error states prevent blank or crashing UI.
- Offline and paused states remain visible and recoverable.

## Security And Stability

- No `v-html` or unsafe DOM injection is used.
- Vue text rendering escapes stream text by default.
- Payloads are type-checked and range-checked before storage.
- Text fields are bounded during validation.
- Timers, subscriptions, and animation frames are cleaned up.
- Dataset and log limits prevent memory leaks during continuous operation.

## Trade-Offs

- The stream is mocked locally rather than backed by a real WebSocket. The service API is intentionally WebSocket-like so it can be replaced later.
- The activity feed uses bounded rendering instead of full virtualization to keep the dependency surface lean while still protecting performance.
- The 1-hour time range works as the stream accumulates data during the session; historical persistence is not included.

## Future Improvements

- Replace the mock stream with a real authenticated WebSocket or Server-Sent Events source.
- Persist user layout, theme, and dataset preferences beyond local theme storage.
- Add chart annotations for incidents and deployments.
- Add command palette and keyboard shortcuts.
- Add draggable dashboard widgets and saved views.
- Add notification routing for critical alerts.
